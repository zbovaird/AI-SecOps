# Latent Space Red Teaming Analysis Report Templates

This directory contains templates for generating comprehensive latent space red teaming analysis reports.

## Template Files

- **`LATENT_SPACE_REDTEAM_ANALYSIS_TEMPLATE.md`** - Main template for latent space red teaming reports
- **Example:** See `../REDTEAM_ANALYSIS_REPORT.md` (if available) for a completed example

## How to Use

1. **Copy the template:**
   ```bash
   cp LATENT_SPACE_REDTEAM_ANALYSIS_TEMPLATE.md YOUR_MODEL_ANALYSIS.md
   ```

2. **Fill in placeholders:**
   - Replace `[PLACEHOLDER]` text with actual values from your analysis
   - Use the example report as a reference for formatting and structure

3. **Key sections to populate:**
   - **Executive Summary:** Overall assessment and key findings
   - **Phase Results:** Detailed findings from each of the 6 phases
   - **Technical Findings:** Why attacks succeeded/failed
   - **Key Findings:** Critical vulnerabilities and model strengths
   - **Recommendations:** Red team and defender action items
   - **Visual Scorecards:** ASCII art visualizations of vulnerability scores

## Data Sources

The template expects data from the following JSON/CSV files (generated by Framework v1):

- `phase1_targets.json` - Layer classifications and compositional kappa
- `gradient_attack_results.json` - FGSM, PGD, BIM, MIM attack results
- `complete_analysis.json` - Full analysis data with all metrics
- `layer_summary.csv` - Per-layer ﾎｺ, ﾏダmin, ﾏダmax values
- `high_value_targets.json` - Priority attack targets
- `attack_summary.json` - Attack results and exploit classifications
- `phase5_reproducibility.json` - Reproducibility test results
- `jacobian_projected_attack_results.json` - SVD-projected attack results

## Report Structure

1. **Executive Summary** - High-level overview with assessment table
2. **Methodology** - Framework architecture and key metrics explained
3. **Detailed Phase Results** - One section per phase (6 total)
4. **Technical Findings** - Why attacks succeeded/failed, Jacobian analysis
5. **Key Findings** - Critical vulnerabilities and strengths
6. **Recommendations** - Red team and defender actions
7. **Visual Scorecards** - ASCII visualizations
8. **Conclusion** - Summary and next steps
9. **Appendix** - Model specs, metrics reference, data file references

## Key Metrics Explained

- **ﾎｺ (Condition Number):** ﾏダmax / ﾏダmin - Measures layer sensitivity
- **ﾏダmax:** Largest singular value - Maximum amplification factor
- **ﾏダmin:** Smallest singular value - Rank deficiency indicator
- **ﾎｺ_comp:** Compositional kappa for MLP blocks - End-to-end sensitivity
- **Semantic Delta:** Embedding similarity change
- **Policy Delta:** Refusal/compliance behavior change
- **Quality Delta:** Coherence/repetition degradation

## Tips

- **Visuals:** Use ASCII art (bar charts, matrices) for GitHub-friendly rendering
- **Risk Levels:** Use emojis (沐ｴ High, 沺｡ Medium, 沺｢ Low, 笨 Safe) for quick visual scanning
- **Data Tables:** Use markdown tables for structured data presentation
- **Code Blocks:** Use code blocks for mathematical equations and attack strategies
- **Layer Names:** Use consistent naming (e.g., `model.layers.7.mlp`)

## Example Usage

After running Framework v1 experiments, generate a report:

```python

import json
import pandas as pd

with open('phase1_targets.json') as f:
    targets = json.load(f)

with open('gradient_attack_results.json') as f:
    attacks = json.load(f)

layer_summary = pd.read_csv('layer_summary.csv')

# Extract key metrics
top_kappa_layers = layer_summary.nlargest(5, 'kappa_comp')
successful_attacks = [a for a in attacks if a['exploit_type'] in ['refusal_bypass', 'policy_violation']]

# Fill in template with these values
```
